เครื่องมือสำหรับผู้พัฒนา — ใช้ด้วยความระมัดระวัง

App Mode (Test / Production)

Test Mode: สามารถลบข้อมูลได้ (DELETE from DB) — ใช้ช่วงพัฒนา
Production Mode: ลบไม่ได้ ทำได้แค่ "ยกเลิก" เพื่อรักษา audit trail

โหมดปัจจุบัน
—

Manager Password

รหัส break-glass สำหรับยืนยันการกระทำ destructive (reset, cancel ใน production) — แยกจาก login password

Seed Demo Data

สร้างข้อมูลตัวอย่าง generic (10 products, 3 suppliers, 3 customers, 2 warehouses, 2 bank accounts, etc.) — เหมาะสำหรับ test flow

DB Statistics

จำนวน rows ในแต่ละตาราง (เรียงมาก→น้อย) — รวมทั้งหมด — rows

Table Rows

Clear Orphan Storage

สแกนไฟล์ใน product-images bucket เทียบกับ URL ที่อ้างถึงใน DB (products.images, company_info.logo_url, po.payment_slip_url, po.receipt_url)
ไฟล์ที่ไม่มีใครอ้าง = orphan → ลบออกเพื่อประหยัดพื้นที่ (ต้อง Manager Password)

Backup / Restore (JSON)

Export: ดาวน์โหลด snapshot ของ DB ทั้งหมดเป็น JSON
Import: กู้คืนจากไฟล์ JSON ที่ export ไว้ (จะลบข้อมูลเดิมทั้งหมดก่อน) — ต้อง Manager Password

⚠️ Danger Zone

การกระทำในส่วนนี้ไม่สามารถกู้คืนได้ ทุกปุ่มต้องยืนยัน Manager Password

Reset Transactional Data Level 1

ลบข้อมูล transactional ทั้งหมด (PO, GR, Return, Movement, Payment, Expense, Sales Orders, etc.)
คงไว้: products, categories, suppliers, customers, warehouses, users, company_info

Reset All Data Level 2

ลบ Transactional + Master data (products, categories, suppliers, warehouses, pages, blocks, promotions, activity log)
คงไว้: users, company_info, app_settings, roles

Factory Reset Level 3 — DANGER

ลบทุกอย่างยกเว้น admin ปัจจุบัน + app_settings + roles
รวมถึง Storage files + reset company info เป็นค่าว่าง
ใช้ในกรณีเริ่มต้นใหม่หมดเท่านั้น
